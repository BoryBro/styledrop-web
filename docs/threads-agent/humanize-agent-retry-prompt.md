# StyleDrop Threads Humanize Retry Prompt

이전 결과는 실패로 본다.

문제:

- 입력 568개 중 300개만 출력했다.
- 기존 글의 의미를 유지하지 않고 너무 일반적인 문장으로 바꿨다.
- 같은 문장이 반복됐다.
- 줄바꿈 없이 문장이 붙어 사람이 쓴 글처럼 보이지 않았다.
- StyleDrop, AI 사진, 카드, 결과물 맥락이 약해졌다.

다시 작업한다.

## 입력 파일

- `styledrop_threads_text_queue_ready.csv`

## 출력 파일

`styledrop_threads_text_queue_humanized_v2.csv`

## 절대 규칙

- 입력 행 수 568개를 반드시 그대로 유지한다.
- `template_id`를 하나도 누락하지 않는다.
- `scheduled_at`을 절대 바꾸지 않는다.
- `status`는 전부 `draft`로 유지한다.
- `category`, `cta_type`, `link_included`, `image_upload_recommended`, `recommended_styles`를 유지한다.
- `post_text`만 재작성한다.
- 500자 초과 금지.
- 같은 문장 반복 금지.
- 줄바꿈을 자연스럽게 유지한다.
- 링크가 있던 글은 링크 유지.
- 링크가 없던 글은 링크 추가 금지.

## 가장 중요한 재작성 원칙

기존 글의 핵심 의미를 버리지 마라.

좋지 않은 방식:

- `해봤는데 결과가 의외로 괜찮더라`
- `나만 이런 생각하나?`
- `그냥 이런 날 있잖아`

이런 식으로 모든 글을 비슷한 일반 문장으로 바꾸면 실패다.

좋은 방식:

- 기존 글에 있던 소재를 살린다.
- StyleDrop/AI 사진/카드/프사/결과물/Before After/친구 반응 같은 맥락을 유지한다.
- 광고 문장만 사람 말투로 낮춘다.
- 너무 매끈한 문장을 조금 자연스럽게 만든다.

## 말투 변형 방식

전체 글에 아래 톤을 섞는다.

- 반말형
- 혼잣말형
- 질문형
- 짧은 후기형
- 제작자 시점
- 친구에게 말하는 톤
- 댓글 유도형

가끔만 섞는다.

- `..`
- `ㅋㅋ`
- `진짜`
- `은근`
- `뭔가`
- `나만 그런가`
- 이모지 1개

주의: 모든 글에 넣지 않는다. 전체의 20~30%만 자연스럽게 사용한다.

## 줄바꿈 규칙

Threads 글처럼 읽히게 줄바꿈을 넣는다.

예:

```text
사진 한 장 넣었는데
분위기가 생각보다 확 바뀌어서 놀람

이런 결과면 프사 바꾸고 싶어질 듯..
```

문장을 한 줄로 붙이지 마라.

## 중복 방지 규칙

출력 전에 반드시 검사한다.

- 완전히 같은 `post_text`가 있으면 안 된다.
- 첫 문장이 5번 이상 반복되면 안 된다.
- `나만 이런 생각하나?`, `해봤는데 결과가`, `그냥 이런 날 있잖아` 같은 문구를 반복하지 않는다.

## 품질 체크

각 글은 아래 질문에 통과해야 한다.

- 기존 글의 소재가 살아있는가?
- 사람이 쓴 것처럼 보이는가?
- 너무 광고 같지 않은가?
- 너무 일반적인 문장으로 바뀌지 않았는가?
- 줄바꿈이 읽기 좋은가?
- 같은 표현이 반복되지 않는가?
- StyleDrop과 연결되는가?
